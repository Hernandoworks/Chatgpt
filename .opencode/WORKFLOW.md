# Workflow

When OpenCode starts, read:

- `.opencode/PROJECT_STATE.md` — current state and next action
- `.opencode/NEXT_TASK.md` — the exact task to execute
- `.opencode/MEMORY.md` — lessons from past work
- `.opencode/KNOWLEDGE_GRAPH.md` — dependency reasoning
- `.opencode/` — remaining instruction files
- `06-docs/project/` — task register, milestones, build summary
- Repository — understand what already exists

Then follow the workflow in `.opencode/WORKFLOWS/standard-task.md`.

If a task requires a component, follow `.opencode/WORKFLOWS/implement-component.md`.

Every 10 completed tasks, run `.opencode/WORKFLOWS/repository-review.md`.

Never skip testing.

Never commit failing code.

Never start the next task without approval.
